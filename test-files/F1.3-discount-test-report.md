## Issues and Bugs Found

### Issue 1: Discount Persistence Between Orders
- **Severity:** Low
- **Description:** When opening payment modal for a new order, the discount from the previous order calculation persisted
- **Example:** Previous order had €8,00 discount; new order opened with €8,00 discount already applied
- **Impact:** User must manually reset discount for each new order
- **Recommendation:** Consider resetting discount to €0,00 when opening payment modal for a new order

### Issue 2: Reason Field Persistence
- **Severity:** Low
- **Description:** Reason text persisted between different orders
- **Example:** "Test complimentary - F1.3 testing" remained in reason field for subsequent orders
- **Impact:** May cause incorrect reason to be recorded
- **Recommendation:** Clear reason field between orders
