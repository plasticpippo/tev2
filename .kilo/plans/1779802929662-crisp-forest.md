# Fix Receipt Feature - Field Name Mismatch in Payment Modal Receipt Service

## Root Cause

In `backend/src/services/paymentModalReceiptService.ts`, the `createReceiptFromPayment` function creates receipt data with **incorrect field names** that don't match what `prepareReceiptTemplateData()` (in `backend/src/types/receipt-template.ts`) and the frontend expect.

### Bug 1: Items snapshot field names

**Current (broken)** at `paymentModalReceiptService.ts:109-116`:
```typescript
const itemsSnapshot = items.map((item: any) => ({
    name: item.name,
    quantity: item.quantity,
    unitPrice: item.price,     // WRONG - downstream expects 'price'
    total: multiplyMoney(...),  // extra field, not expected
    taxRateName: ...,
    taxRatePercent: ...,
}));
```

**Expected** by `prepareReceiptTemplateData()` at `receipt-template.ts:131`:
```typescript
items: receipt.itemsSnapshot.map(item => ({
    price: Number(item.price),  // accesses 'price' â†’ gets undefined â†’ NaN
    ...
}))
```

Result: All receipt item prices render as `NaN` in PDFs and frontend previews.

### Bug 2: Tax breakdown field names

**Current (broken)** at `paymentModalReceiptService.ts:156`:
```typescript
taxBreakdown.push({
    name,          // WRONG - should be 'rateName'
    percent,       // WRONG - should be 'ratePercent'
    taxableAmount,
    taxAmount,
});
```

**Expected** by `ReceiptTemplateTaxBreakdown` type and template:
```typescript
{
    rateName: string,
    ratePercent: number,
    taxableAmount: number,
    taxAmount: number,
}
```

Result: Tax breakdown shows empty/undefined labels and percentages in PDFs.

## Impact

| Flow | Affected? | Symptom |
|------|-----------|---------|
| Receipt from payment modal (immediate) | YES | PDF shows NaN for all prices, broken tax labels |

## Fix

### Single file change: `backend/src/services/paymentModalReceiptService.ts`

#### Change 1: Fix items snapshot (lines 109-116)

```typescript
// BEFORE:
const itemsSnapshot = items.map((item: any) => ({
    name: item.name,
    quantity: item.quantity,
    unitPrice: item.price,
    total: multiplyMoney(Number(item.price), item.quantity),
    taxRateName: item.taxRateName || 'Standard',
    taxRatePercent: item.taxRatePercent ?? Math.round((item.effectiveTaxRate || 0.22) * 100),
}));

// AFTER:
const itemsSnapshot = items.map((item: any) => ({
    id: item.id,
    variantId: item.variantId,
    productId: item.productId,
    name: item.name,
    price: Number(item.price),
    quantity: item.quantity,
    taxRateId: item.taxRateId,
    taxRateName: item.taxRateName || 'Standard',
    taxRatePercent: item.taxRatePercent ?? Math.round((item.effectiveTaxRate || 0.22) * 100),
}));
```

This aligns with the `ReceiptItemSnapshot` type in `backend/src/types/receipt.ts:52-62` and matches what `prepareReceiptTemplateData` expects.

#### Change 2: Fix tax breakdown (lines 154-162)

```typescript
// BEFORE:
taxMap.forEach((value, key) => {
    const [name, percentStr] = key.split('-');
    taxBreakdown.push({
        name,
        percent: parseInt(percentStr, 10),
        taxableAmount: roundMoney(value.taxableAmount),
        taxAmount: roundMoney(value.taxAmount),
    });
});

// AFTER:
taxMap.forEach((value, key) => {
    const [name, percentStr] = key.split('-');
    taxBreakdown.push({
        rateName: name,
        ratePercent: parseInt(percentStr, 10),
        taxableAmount: roundMoney(value.taxableAmount),
        taxAmount: roundMoney(value.taxAmount),
    });
});
```

This aligns with the `TaxBreakdownItem` type in `shared/types.ts:339-344` and `ReceiptTemplateTaxBreakdown` in `receipt-template.ts:35-40`.
