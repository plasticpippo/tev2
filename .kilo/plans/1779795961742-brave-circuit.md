# Rename App to AssoPOS - Comprehensive Plan

## Overview

The app currently uses multiple names across the codebase: **"Bar POS Pro"**, **"Bar POS"**, **"TEV2"**, **"TotalEVO"**, and **"bar_pos"**. All display names, Docker container names, package names, and internal references must be replaced with **"AssoPOS"**.

**IMPORTANT**: The database name (`bar_pos`) and credentials (`totalevo_user`/`totalevo_password`) will be **kept as-is** to preserve existing data. Only defaults in `.env.example` and `docker-compose.yml` fallback values will be updated for new installations.

### Naming Conventions

| Context | Old | New |
|---------|-----|-----|
| Display name (UI, titles) | Bar POS Pro / Bar POS | AssoPOS |
| npm package name | bar-pos-* | assopos-* |
| Docker container name | bar_pos_* | assopos_* |
| Docker label | bar-pos-* | assopos-* |
| Backup file prefix | tev2_full_ | assopos_full_ |
| Cloud backup path | /TEV2/backups | /AssoPOS/backups |
| Installer/internal | TEV2 | AssoPOS |
| DB name (existing .env) | bar_pos | bar_pos (unchanged) |
| DB creds (existing .env) | totalevo_user/totalevo_password | unchanged |

## Phase 10: Post-Change Actions

1. **Regenerate package-lock.json files:**
   ```bash
   npm install
   cd backend && npm install
   cd ../frontend && npm install
   cd ../mega-sidecar && npm install
   ```

2. **Rebuild Docker containers:**
   ```bash
   docker compose down
   docker compose up -d --build
   ```
   Note: Since `.env` keeps existing DB values, data is preserved. Container names change but volumes persist.

3. **Verify:**
   - App loads at http://192.168.1.70
   - Page title shows "AssoPOS"
   - Login screen shows "AssoPOS"
   - `/api/version` returns `name: "assopos-backend"`
   - All containers healthy: `docker compose ps`

## Risk Notes

1. **Container name change**: `docker compose down` + `up` handles volume remapping. Existing data in `postgres_data` volume persists since volume names don't change.
2. **Backup file format**: Existing `tev2_full_*` backups won't auto-match new `assopos_full_*` pattern. Old backups can still be manually restored.
3. **Package-lock.json**: Must be regenerated to avoid npm install errors.
4. **`.env` values preserved**: The live `.env` and `backend/.env` keep existing DB name/credentials, so no data migration needed.
5. **`docker-compose.yml` defaults changed**: New installations (using defaults) will get `assopos` as DB name and `assopos_user` as user. Existing `.env` overrides these defaults.
