    User Management

    User accounts allow staff to access the POS system. This guide explains how to create, edit, and manage user accounts, including role assignment and password management.

    Viewing All Users

    Navigate to the Users section from the Admin Panel sidebar to see a complete list of all user accounts in your system.

    User List Information

    For each user, the list displays:

    • Name: The user's display name
    • Username: The login identifier
    • Role: Admin or Cashier
    • Actions: Edit and delete buttons

    Adding New Users

    To create a new user account:

    1. Click the Add User button at the top of the user list
    2. Enter the user's Name (display name shown in the interface)
    3. Enter a Username (used for login)
    4. Enter a Password for the account
    5. Select a Role (Admin or Cashier)
    6. Click Save to create the account

    Username Requirements

    • Must be unique (no two users can have the same username)
    • Case-insensitive (John and john are considered the same)
    • Should be easy to remember for the user
    • Cannot be changed after creation (without deleting and recreating)

    Password Requirements

    • Should be at least 8 characters long
    • Should include a mix of letters, numbers, and symbols
    • Passwords are securely hashed and stored encrypted
    Security Note: Choose strong passwords for admin accounts. Admin users have full access to the system, including sensitive financial data.

    User Roles and Permissions

    The POS system uses role-based access control to manage what users can do. There are two primary roles:

    Role Permissions Typical Use
    Admin
    • Full access to Admin Panel
    • Create, edit, delete products
    • Manage users and roles
    • Configure system settings
    • Close business day
    • View all reports and analytics
    • Manage inventory and stock
    • All Cashier permissions
    		 Managers, owners, supervisors
    Cashier
    • Access POS interface only
    • Process orders and payments
    • Create and manage tabs
    • Assign tables
    • No Admin Panel access
    		 Cashiers, servers, bartenders

    Assigning Roles

    When creating a user, select the appropriate role based on their job responsibilities:

    • Admin: For users who need to manage the system, configure settings, or view reports
    • Cashier: For users who only need to process sales and manage orders
    Best Practice: Limit admin access to only those who truly need it. The more admin accounts, the higher the security risk.

    Editing User Accounts

    To modify an existing user account:

    1. Locate the user in the user list
    2. Click the Edit button (pencil icon)
    3. Modify the desired fields:
      • Name: Update the display name
      • Role: Change between Admin and Cashier
      • Password: Enter a new password (optional)
    4. Click Save to apply changes

    What Can Be Changed

    • Name: Can be changed at any time
    • Role: Can be upgraded or downgraded as needed
    • Password: Can be reset (see Password Management section)
    • Username: Cannot be changed directly; create a new user instead

    Password Management

    Passwords are a critical security component. The system provides tools for administrators to manage user passwords.

    Resetting a User's Password

    As an administrator, you can reset any user's password:

    1. Edit the user account
    2. Enter a new password in the Password field
    3. Save the changes
    4. Communicate the new password to the user securely
    Note: When you reset a password, the user's existing sessions will remain active until they log out or the session expires. For immediate security concerns, consider revoking all tokens.

    Password Security Best Practices

    • Use strong, unique passwords for each account
    • Change passwords periodically
    • Never share passwords between users
    • Change passwords immediately if you suspect they've been compromised
    • Communicate new passwords through secure channels (not email or text)

    Revoking User Sessions

    If you need to immediately log out a user from all devices:

    1. This action revokes all authentication tokens for the user
    2. The user will need to log in again with their current password
    3. Use this if you suspect unauthorized access or when an employee leaves

    Deleting User Accounts

    To remove a user account from the system:

    1. Locate the user in the user list
    2. Click the Delete button (trash icon)
    3. Confirm the deletion when prompted
    Before Deleting: Consider the following:
    • Historical transactions will retain the user's name
    • Daily closings will still reference the user who performed them
    • You cannot recover a deleted account

    Alternative to Deletion

    Instead of deleting a user, consider:

    • Changing their role to Cashier if they no longer need admin access
    • Changing their password to prevent login while preserving records

    Viewing User Performance

    The system tracks user activity for performance analysis. You can view individual user statistics:

    Available Metrics

    • Transactions Processed: Number of sales completed
    • Total Sales: Revenue generated by the user
    • Average Transaction: Average sale amount
    • Items Removed: Track of cleared items (for accountability)

    Accessing User Reports

    1. Go to the Users section
    2. Click on a user's name or the View Report button
    3. Select the date range for the report
    4. Review the performance metrics

    Security Considerations

    User management is a critical security function. Follow these guidelines to maintain system security:

    Account Security

    • Minimum Privilege: Grant the lowest role necessary for each user's duties
    • Regular Audits: Periodically review user accounts and remove unnecessary ones
    • Strong Passwords: Enforce strong password practices
    • Immediate Action: Disable or delete accounts immediately when employees leave

    Admin Account Protection

    • Limit the number of admin accounts
    • Use unique, strong passwords for each admin
    • Never share admin credentials
    • Log out when leaving the workstation unattended
    Critical: If an admin account is compromised, immediately change the password and review all recent activity for unauthorized changes.
