Introduction

Welcome to the POS (Point of Sale) application documentation. This comprehensive guide provides detailed information about the system for both end users (cashiers and staff) and administrators (managers and owners).

What is the POS System?

The POS (Point of Sale) application is a modern, web-based system designed specifically for restaurants, cafes, bars, and similar hospitality businesses. It provides a complete solution for managing customer orders, table assignments, payment processing, inventory tracking, and business analytics.

Built with a responsive design, the system works seamlessly on desktop computers, tablets, and touchscreen devices, making it ideal for various hospitality environments from small cafes to large restaurants with multiple service areas.

Key Features

The POS system provides a comprehensive set of features organized by functional area:

Order Management

  • Product Grid - Visual grid layout of products organized by category with customizable button colors
  • Product Variants - Support for product variations (e.g., Small, Medium, Large; Glass, Bottle)
  • Favourites - Quick access to frequently ordered items marked with a star
  • Quantity Adjustment - Easy +/- buttons to adjust item quantities in orders
  • Order Clearing - Clear entire orders with confirmation to prevent accidental loss

Table & Room Management

  • Visual Floor Plan - Drag-and-drop layout editor for tables and rooms
  • Table Status - Track table availability (Available, Occupied, Reserved, Unavailable)
  • Room Organization - Group tables by service areas (e.g., Main Dining, Patio, Bar)
  • Table Assignment - Assign orders to specific tables for service tracking

Payment Processing

  • Multiple Payment Methods - Support for Cash and Card payments
  • Tip Handling - Add tips to transactions with automatic calculation
  • Tax Configuration - Flexible tax modes (inclusive, exclusive, or no tax)
  • Transaction History - Complete record of all transactions with filtering

Tab Management

  • Open Tabs - Save orders for later payment (ideal for bar tabs)
  • Tab Loading - Load existing tabs to add items or process payment
  • Item Transfer - Move items between open tabs
  • Tab Closing - Close tabs with full payment processing

Inventory & Stock

  • Stock Items - Manage ingredients and sellable goods
  • Stock Adjustments - Record stock additions and removals
  • Consumption Tracking - Automatic tracking of ingredient consumption per sale
  • Itemised Consumption Reports - View stock usage by time period

Analytics & Reporting

  • Sales Dashboard - Real-time view of daily sales totals
  • Hourly Charts - Visualize sales patterns throughout the day
  • Top Products - Identify best-selling items
  • Daily Closing - End-of-day financial summaries with payment breakdown
  • User Performance - Track individual staff member statistics

System Features

  • Multi-language Support - Available in English and Italian
  • Virtual Keyboard - On-screen keyboard optimized for touch devices
  • Multi-Till Support - Multiple POS terminals with independent product visibility
  • User Roles - Admin and Cashier roles with appropriate access levels

Target Users

The POS system is designed for two primary user roles, each with specific capabilities:

Cashiers / Staff

Restaurant staff who interact with the system daily to process customer orders and handle payments. Cashiers have access to the POS interface with the following capabilities:

  • Take and modify customer orders
  • Assign orders to tables
  • Process cash and card payments
  • Create and manage open tabs
  • Switch between product categories
  • Use the virtual keyboard for touch input

Note: Cashiers must be assigned to a configured till before they can log in and use the system.

Administrators / Managers

Restaurant managers and owners who manage the system configuration, products, users, inventory, and view business analytics. Administrators have full access to:

  • Dashboard - Overview of sales, tills, and open tabs
  • Product Management - Add, edit, and delete products and variants
  • Category Management - Organize products and control till visibility
  • User Management - Create users, assign roles, reset passwords
  • Till Management - Create and configure POS terminals
  • Table Management - Design floor plans with rooms and tables
  • Stock Items - Manage inventory items and track stock levels
  • Transaction History - View and filter past transactions
  • Analytics - Sales trends and performance reports
  • Daily Closing - Close business days and generate reports
  • Settings - Configure tax rates and business day timing

Special Access: Administrators can log in even when a till is not configured, allowing them to set up the system initially.

System Architecture

The POS application follows a modern three-tier web application architecture with separate frontend, backend, and database components. This design ensures scalability, maintainability, and the ability to deploy each component independently.

Component Technology Description
Frontend React / TypeScript Web-based user interface with responsive design
Backend Node.js / Express REST API server handling business logic
Database PostgreSQL Persistent data storage with Prisma ORM
Authentication JWT Tokens Secure user authentication with token blacklist
Internationalization i18next Multi-language support (English, Italian)

Key Architecture Features

  • Containerized Deployment - Docker containers for easy deployment and scaling
  • API-First Design - RESTful API enables integration with other systems
  • Real-time Updates - Data synchronization across multiple terminals
  • Secure Authentication - JWT tokens with blacklist for secure logout
