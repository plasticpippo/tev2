    System Requirements

    This page outlines the hardware, software, and network requirements needed to run the POS application effectively. Requirements are organized by deployment scenario.

    Hardware Requirements

    The POS system can run on various devices, from desktop computers to tablets. Choose hardware based on your service environment:

    POS Terminal Devices

    Component Minimum Recommended
    Display Size 10 inches 15+ inches (touchscreen recommended)
    Screen Resolution 1024 x 768 1920 x 1080 (Full HD)
    Processor Dual Core 2GHz Quad Core 3GHz+
    Memory (RAM) 4 GB 8 GB+
    Storage 10 GB free 20 GB+ SSD
    Touch Screen Optional Highly recommended for fast order entry

    Recommended Hardware by Environment

    Restaurant / Bar
    • 15+ inch touchscreen terminal
    • Dedicated POS stand or mount
    • Cash drawer connected via USB or network
    • Receipt printer (thermal, 80mm recommended)
    • Card reader for payment processing
    Cafe / Quick Service
    • Tablet (10+ inch) with stand
    • Portable Bluetooth receipt printer
    • Mobile card reader
    • Optional: Kitchen display system

    Browser Compatibility

    The POS application is a modern web application that requires an up-to-date browser. For the best experience, we recommend:

    Browser Minimum Version Status Notes
    Google Chrome 80+ Recommended Best performance and touch support
    Mozilla Firefox 75+ Supported Full feature support
    Microsoft Edge 80+ Supported Chromium-based versions
    Safari (macOS) 13+ Limited Some touch features may differ
    Safari (iOS) iOS 13+ Limited Works on iPad, best in landscape mode
    Important

    Internet Explorer is NOT supported. Please use a modern browser for the best experience.

    For touch devices, we recommend Chrome or Edge for optimal virtual keyboard and touch interaction.

    Browser Settings

    For optimal POS operation, ensure the following browser settings:

    • JavaScript - Must be enabled (required)
    • Cookies - Must be enabled for session management
    • Pop-up Blocker - Disable or allow pop-ups from the POS domain
    • Cache - Clear cache if experiencing display issues after updates

    Network Requirements

    A stable network connection is essential for reliable POS operation. The system can be deployed in two network configurations:

    Local Network (LAN) Deployment

    Recommended for restaurants with multiple terminals:

    • Connection Type - Wired Ethernet preferred; WiFi acceptable for tablets
    • Bandwidth - Minimum 10 Mbps for the local network
    • Latency - Under 50ms between terminals and server
    • Network Ports:
      • Port 3000 - Frontend application
      • Port 3001 - Backend API server
      • Port 5432 - PostgreSQL database (internal only)

    Internet Access Requirements

    • Minimum Speed - 5 Mbps download / 2 Mbps upload
    • Recommended Speed - 25+ Mbps for multiple terminals
    • Latency - Under 100ms to server for responsive operation
    • Firewall - Ports 80 (HTTP) and 443 (HTTPS) must be accessible
    Offline Operation
    The POS system requires an active connection to the backend server. If network connectivity is lost, terminals will not be able to process orders or payments. Consider implementing a backup network connection for critical service periods.

    Database Requirements

    The POS system uses PostgreSQL for data storage. Database requirements depend on your deployment:

    Requirement Minimum Recommended
    PostgreSQL Version 13.0 15+ (latest stable)
    Disk Space 500 MB 5 GB+ (depends on transaction volume)
    Memory (RAM) 512 MB allocated 2 GB+ allocated
    Max Connections 20 100+ (for multiple terminals)

    Database Configuration

    The database is configured via the DATABASE_URL environment variable:

    DATABASE_URL="postgresql://username:password@hostname:5432/database_name"

    Docker Deployment (Recommended)

    The POS system is designed for containerized deployment using Docker:

    • Docker Engine - Version 20.10+
    • Docker Compose - Version 2.0+
    • Containers:
      • Frontend container (Nginx serving React app)
      • Backend container (Node.js/Express API)
      • Database container (PostgreSQL)
    Docker Benefits
    Docker deployment provides isolated environments, easy updates, consistent configuration, and simplified scaling across multiple terminals.

    Security Requirements

    Ensure the following security measures are in place:

    Authentication Security

    • JWT Secret - Use a strong, unique secret key (minimum 32 characters)
    • Password Hashing - Passwords are automatically hashed using bcrypt
    • Session Management - Tokens are blacklisted on logout

    Network Security

    • HTTPS - Use SSL/TLS certificates for production deployments
    • Firewall - Restrict database port (5432) to internal network only
    • Rate Limiting - Built-in rate limiting protects against brute force attacks
    Default Credentials

    The system ships with default credentials:

    • Admin: username: admin, password: admin123
    • Cashier: username: cashier, password: cashier123

    Change these passwords immediately after installation!
