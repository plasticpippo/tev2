    Requisiti di Sistema

    Questa pagina descrive i requisiti hardware, software e di rete necessari per eseguire l'applicazione POS in modo efficace. I requisiti sono organizzati per scenario di distribuzione.

    Dispositivi Terminale POS

    Componente Minimo Consigliato
    Dimensione Display 10 pollici 15+ pollici (touchscreen consigliato)
    Risoluzione Schermo 1024 x 768 1920 x 1080 (Full HD)
    Processore Dual Core 2GHz Quad Core 3GHz+
    Memoria (RAM) 4 GB 8 GB+
    Archiviazione 10 GB liberi 20 GB+ SSD

    Ristorante / Bar
    • Terminale touchscreen 15+ pollici

    Compatibilità Browser

    L'applicazione POS è un'applicazione web moderna che richiede un browser aggiornato. Per la migliore esperienza, consigliamo:

    Browser Versione Minima Stato Note
    Google Chrome 80+ Consigliato Migliori performance e supporto touch
    Mozilla Firefox 75+ Supportato Supporto completo funzionalità
    Microsoft Edge 80+ Supportato Versioni basate su Chromium
    Safari (macOS) 13+ Limitato Alcune funzionalità touch potrebbero differire
    Safari (iOS) iOS 13+ Limitato Funziona su iPad, migliore in modalità orizzontale
    Importante

    Internet Explorer NON è supportato. Per la migliore esperienza, usa un browser moderno.

    Per dispositivi touch, consigliamo Chrome o Edge per interazione ottimale con tastiera virtuale e touch.

    Impostazioni Browser

    Per un funzionamento ottimale del POS, assicurati delle seguenti impostazioni del browser:

    • JavaScript - Deve essere abilitato (richiesto)
    • Cookie - Devono essere abilitati per la gestione della sessione
    • Blocco Popup - Disabilita o consenti popup dal dominio POS
    • Cache - Svuota la cache se riscontri problemi di visualizzazione dopo aggiornamenti

    Requisiti di Rete

    Una connessione di rete stabile è essenziale per un funzionamento affidabile del POS. Il sistema può essere distribuito in due configurazioni di rete:

    Distribuzione in Rete Locale (LAN)

    Consigliato per ristoranti con terminali multipli:

    • Tipo di Connessione - Ethernet cablata preferita; WiFi accettabile per tablet
    • Larghezza Banda - Minimo 10 Mbps per la rete locale
    • Latenza - Sotto 50ms tra terminali e server
    • Porte di Rete:
      • Porta 3000 - Applicazione Frontend
      • Porta 3001 - Server API Backend
      • Porta 5432 - Database PostgreSQL (solo interno)

    Requisiti Accesso Internet

    • Velocità Minima - 5 Mbps download / 2 Mbps upload
    • Velocità Consigliata - 25+ Mbps per terminali multipli
    • Latenza - Sotto 100ms al server per un funzionamento reattivo
    • Firewall - Le porte 80 (HTTP) e 443 (HTTPS) devono essere accessibili
    Funzionamento Offline
    Il sistema POS richiede una connessione attiva al server backend. Se la connettività di rete viene persa, i terminali non saranno in grado di elaborare ordini o pagamenti. Considera di implementare una connessione di rete backup per periodi di servizio critici.

    Requisiti Database

    Il sistema POS utilizza PostgreSQL per l'archiviazione dei dati. I requisiti del database dipendono dalla tua distribuzione:

    Requisito Minimo Consigliato
    Versione PostgreSQL 13.0 15+ (ultima stabile)
    Spazio su Disco 500 MB 5 GB+ (dipende dal volume delle transazioni)
    Memoria (RAM) 512 MB allocati 2 GB+ allocati
    Connessioni Max 20 100+ (per terminali multipli)

    Configurazione Database

    Il database è configurato tramite la variabile di ambiente DATABASE_URL:

    DATABASE_URL="postgresql://username:password@hostname:5432/database_name"

    Distribuzione Docker (Consigliata)

    Il sistema POS è progettato per la distribuzione containerizzata usando Docker:

    • Docker Engine - Versione 20.10+
    • Docker Compose - Versione 2.0+
    • Container:
      • Container Frontend (Nginx che serve app React)
      • Container Backend (Node.js/Express API)
      • Container Database (PostgreSQL)
    Vantaggi Docker
    La distribuzione Docker fornisce ambienti isolati, aggiornamenti facili, configurazione coerente e scalabilità semplificata su terminali multipli.

    Requisiti di Sicurezza

    Assicurati che le seguenti misure di sicurezza siano in atto:

    Sicurezza Autenticazione

    • JWT Secret - Usa una chiave segreta forte e unica (minimo 32 caratteri)
    • Hashing Password - Le password vengono automaticamente hashate usando bcrypt
    • Gestione Sessione - I token vengono inseriti nella blacklist al logout

    Sicurezza di Rete

    • HTTPS - Usa certificati SSL/TLS per distribuzioni di produzione
    • Firewall - Limita la porta del database (5432) solo alla rete interna
    • Rate Limiting - Il limitatore di velocità integrato protegge contro attacchi brute force
    Credenziali Predefinite

    Il sistema viene fornito con credenziali predefinite:

    • Admin: username: admin, password: admin123
    • Cassiere: username: cashier, password: cashier123

    Cambia queste password immediatamente dopo l'installazione!
