## Timeout Handling Strategy Documentation

### Frontend Strategy
1. **AbortController Pattern:** All API requests use AbortController with configurable timeout
2. **Error Translation:** Timeout errors are translated to user-friendly messages via i18n
3. **State Preservation:** Order state is maintained in React state, not cleared on timeout
4. **Request Deduplication:** Cache mechanism prevents duplicate requests during timeout scenarios

### Backend Strategy
1. **Connection Pooling:** Prisma manages connection pool with configurable limits
2. **Automatic Reconnection:** Database disconnections trigger automatic retry (up to 10 attempts)
3. **Health Checks:** `/api/health` endpoint verifies database connectivity
4. **Graceful Degradation:** App continues running even if initial DB connection fails

### Recovery Mechanisms
1. **Frontend:** User can retry after timeout - UI remains responsive
2. **Backend:** Background reconnection attempts allow app to start even with DB issues
3. **Database:** Connection state tracking enables automatic recovery when DB becomes available
